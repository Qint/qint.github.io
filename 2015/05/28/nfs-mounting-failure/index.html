NFS挂载失败

Posted on 2015-05-28   |  

收到同学反馈说：挂载其他节点的目录作为NFS文件系统失败。立即查看情况。问题如下：

在客户机节点上使用root用户挂载server节点上的/home/xxx目录；命令函错误提示是mount.nfs access denied by server while mounting…….

感觉应该不是权限问题。根据网上的提示，在目标机器上输出日志查看更详细的错误提示；

在最后面看到了最近的几条关于mount相关的错误信息

得到的进一步错误提示是no export entry。根据提示，查看/etc/exports文件中的信息：

以上命令输出NFS服务器允许挂载的目录。查看到，挂载命令想要挂载的目标目录不在配置文件之中。

解决办法：vim /etc/exports编辑配置文件，复制一行，修改开头的目标目录为/home/xxx。重启NFS服务：

再测试，依然失败。cat /var/log/messages | grep mount没有看到输出信息，并且restart NFS服务的时候，有一些failed信息。经过一番搜索，终于找到如下的解决方案。

解决方法可能与具体的环境有关，总结要点：

  1. 挂载NFS需要在目标服务器上修改配置，允许目标目录被挂载，同时设置限制的访问IP和权限；
  2. 修改完配置文件时后需要重启服务：将rpcbind和nfs两个服务关掉，并按顺序启动；
  3. 启动上述服务的时候，需要关闭防火墙；
